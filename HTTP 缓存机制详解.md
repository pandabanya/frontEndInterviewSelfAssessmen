# HTTP 缓存机制详解

## 什么是 HTTP 缓存？

HTTP 缓存是浏览器或服务器为了提高网页加载速度而采取的一种机制。当用户访问一个网页时，浏览器会将网页的资源（如 HTML、CSS、JS、图片等）存储在本地，下次访问时可以直接从本地读取，而不需要重新从服务器下载，从而提高了网页的加载速度。

## 缓存两个大类型

### 强缓存

比较强势哈，不发送到服务器，直接从本地读取 200

相关头部
Expire 值类型：绝对时间 但是这个依赖客户端时间，不准
Cache-Control 值类型：相对时间 比如max-age = 3600 缓存3600s

### 协商缓存

发请求到服务器，让服务器决定是否使用缓存，服务器说没变那就用缓存，说变了那就重新下载 304。

当强混存过期了失效了，那就用协商缓存。

Last-Modified / If-Modified-Since
首次请求 → 服务器返回 Last-Modified: 资源最后修改时间
再次请求 → 浏览器带上 If-Modified-Since: 上次的时间
服务器对比时间 → 没变则 304
缺点： 精度只到秒级，文件重新保存（内容没变）时间戳也会变。

ETag / If-None-Match
首次请求 → 服务器返回 ETag: "abc123"（内容的 hash）
再次请求 → 浏览器带上 If-None-Match: "abc123"
服务器对比 ETag → 一致则 304
优先级高于 Last-Modified，解决了上述缺点。

```plaintxt
浏览器请求资源
  │
  ▼
有缓存？──否──→ 向服务器请求，拿到资源 + 缓存标识
  │是
  ▼
强缓存是否过期？（看 Cache-Control / Expires）
  │
  ├─ 未过期 → 200 (from cache)，不发请求 ✅
  │
  └─ 已过期 → 走协商缓存
                │
                ▼
         携带 If-None-Match / If-Modified-Since 请求服务器
                │
                ├─ 资源没变 → 304，用本地缓存 ✅
                │
                └─ 资源变了 → 200，返回新资源 + 新缓存标识
```

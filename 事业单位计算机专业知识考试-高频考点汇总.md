## 二、数据库SQL语句(重点!)

### 1. 基础查询(SELECT)

#### 1.1 简单查询
```sql
-- 查询所有列
SELECT * FROM 学生表;

-- 查询指定列
SELECT 姓名, 年龄, 性别 FROM 学生表;

-- 查询并去重
SELECT DISTINCT 专业 FROM 学生表;

-- 查询并重命名列
SELECT 姓名 AS name, 年龄 AS age FROM 学生表;
```

#### 1.2 条件查询(WHERE)
```sql
-- 等于
SELECT * FROM 学生表 WHERE 年龄 = 20;

-- 不等于
SELECT * FROM 学生表 WHERE 性别 != '男';
SELECT * FROM 学生表 WHERE 性别 <> '男';

-- 大于、小于
SELECT * FROM 学生表 WHERE 年龄 > 18;
SELECT * FROM 学生表 WHERE 年龄 >= 18;

-- 范围查询(BETWEEN)
SELECT * FROM 学生表 WHERE 年龄 BETWEEN 18 AND 25;

-- 多个值(IN)
SELECT * FROM 学生表 WHERE 专业 IN ('计算机', '软件工程');

-- 模糊查询(LIKE)
SELECT * FROM 学生表 WHERE 姓名 LIKE '张%';    -- 张开头
SELECT * FROM 学生表 WHERE 姓名 LIKE '%明%';  -- 包含明
SELECT * FROM 学生表 WHERE 姓名 LIKE '张_';   -- 张+一个字符

-- 空值查询
SELECT * FROM 学生表 WHERE 电话 IS NULL;
SELECT * FROM 学生表 WHERE 电话 IS NOT NULL;

-- 逻辑运算(AND、OR、NOT)
SELECT * FROM 学生表 WHERE 年龄 > 18 AND 性别 = '男';
SELECT * FROM 学生表 WHERE 专业 = '计算机' OR 专业 = '软件工程';
SELECT * FROM 学生表 WHERE NOT 性别 = '男';
```

#### 1.3 排序(ORDER BY)
```sql
-- 升序(默认)
SELECT * FROM 学生表 ORDER BY 年龄;
SELECT * FROM 学生表 ORDER BY 年龄 ASC;

-- 降序
SELECT * FROM 学生表 ORDER BY 年龄 DESC;

-- 多列排序
SELECT * FROM 学生表 ORDER BY 专业 ASC, 年龄 DESC;
```

#### 1.4 限制结果(LIMIT/TOP)
```sql
-- MySQL/PostgreSQL
SELECT * FROM 学生表 LIMIT 10;           -- 前10条
SELECT * FROM 学生表 LIMIT 10 OFFSET 20; -- 跳过20条,取10条

-- SQL Server
SELECT TOP 10 * FROM 学生表;
```

### 2. 聚合函数

```sql
-- 计数
SELECT COUNT(*) FROM 学生表;
SELECT COUNT(DISTINCT 专业) FROM 学生表;

-- 求和
SELECT SUM(成绩) FROM 成绩表;

-- 平均值
SELECT AVG(成绩) FROM 成绩表;

-- 最大值、最小值
SELECT MAX(成绩) FROM 成绩表;
SELECT MIN(成绩) FROM 成绩表;
```

### 3. 分组查询(GROUP BY)

```sql
-- 按专业分组,统计人数
SELECT 专业, COUNT(*) AS 人数 
FROM 学生表 
GROUP BY 专业;

-- 按专业分组,计算平均年龄
SELECT 专业, AVG(年龄) AS 平均年龄 
FROM 学生表 
GROUP BY 专业;

-- 分组后筛选(HAVING)
SELECT 专业, COUNT(*) AS 人数 
FROM 学生表 
GROUP BY 专业 
HAVING COUNT(*) > 10;

-- WHERE vs HAVING
-- WHERE: 分组前筛选
-- HAVING: 分组后筛选
SELECT 专业, AVG(年龄) AS 平均年龄
FROM 学生表
WHERE 性别 = '男'           -- 先筛选男生
GROUP BY 专业               -- 再分组
HAVING AVG(年龄) > 20;      -- 最后筛选平均年龄>20的组
```

### 4. 多表查询(JOIN)

#### 4.1 内连接(INNER JOIN)
```sql
-- 查询学生及其成绩
SELECT 学生表.姓名, 成绩表.课程, 成绩表.分数
FROM 学生表
INNER JOIN 成绩表 ON 学生表.学号 = 成绩表.学号;

-- 简写(省略INNER)
SELECT s.姓名, c.课程, c.分数
FROM 学生表 s
JOIN 成绩表 c ON s.学号 = c.学号;
```

#### 4.2 左连接(LEFT JOIN)
```sql
-- 查询所有学生及其成绩(包括没有成绩的学生)
SELECT s.姓名, c.课程, c.分数
FROM 学生表 s
LEFT JOIN 成绩表 c ON s.学号 = c.学号;
```

#### 4.3 右连接(RIGHT JOIN)
```sql
-- 查询所有成绩及对应学生(包括没有学生信息的成绩)
SELECT s.姓名, c.课程, c.分数
FROM 学生表 s
RIGHT JOIN 成绩表 c ON s.学号 = c.学号;
```

#### 4.4 全连接(FULL JOIN)
```sql
-- 查询所有学生和所有成绩
SELECT s.姓名, c.课程, c.分数
FROM 学生表 s
FULL JOIN 成绩表 c ON s.学号 = c.学号;
```

### 5. 子查询

```sql
-- 查询成绩高于平均分的学生
SELECT 姓名, 分数
FROM 成绩表
WHERE 分数 > (SELECT AVG(分数) FROM 成绩表);

-- IN子查询
SELECT 姓名
FROM 学生表
WHERE 学号 IN (SELECT 学号 FROM 成绩表 WHERE 分数 > 90);

-- EXISTS子查询
SELECT 姓名
FROM 学生表 s
WHERE EXISTS (SELECT 1 FROM 成绩表 c WHERE c.学号 = s.学号 AND c.分数 > 90);
```

### 6. 插入数据(INSERT)

```sql
-- 插入完整记录
INSERT INTO 学生表 (学号, 姓名, 年龄, 性别)
VALUES ('001', '张三', 20, '男');

-- 插入多条记录
INSERT INTO 学生表 (学号, 姓名, 年龄, 性别)
VALUES 
  ('002', '李四', 21, '男'),
  ('003', '王五', 19, '女');

-- 从其他表插入
INSERT INTO 学生备份表
SELECT * FROM 学生表 WHERE 年龄 > 18;
```

### 7. 更新数据(UPDATE)

```sql
-- 更新单条记录
UPDATE 学生表 SET 年龄 = 21 WHERE 学号 = '001';

-- 更新多个字段
UPDATE 学生表 
SET 年龄 = 21, 专业 = '软件工程' 
WHERE 学号 = '001';

-- 批量更新
UPDATE 学生表 SET 年龄 = 年龄 + 1 WHERE 专业 = '计算机';
```

### 8. 删除数据(DELETE)

```sql
-- 删除指定记录
DELETE FROM 学生表 WHERE 学号 = '001';

-- 删除所有记录(保留表结构)
DELETE FROM 学生表;

-- 清空表(更快,不能回滚)
TRUNCATE TABLE 学生表;
```

### 9. 表操作(DDL)

```sql
-- 创建表
CREATE TABLE 学生表 (
  学号 VARCHAR(10) PRIMARY KEY,
  姓名 VARCHAR(50) NOT NULL,
  年龄 INT,
  性别 CHAR(1),
  专业 VARCHAR(50),
  入学日期 DATE
);

-- 修改表结构
ALTER TABLE 学生表 ADD 电话 VARCHAR(20);        -- 添加列
ALTER TABLE 学生表 DROP COLUMN 电话;            -- 删除列
ALTER TABLE 学生表 MODIFY 年龄 SMALLINT;        -- 修改列类型

-- 删除表
DROP TABLE 学生表;
```

## 三、数据库设计(三范式)

### 第一范式(1NF): 列不可分

**要求:** 每一列都是不可分割的原子数据项

```
❌ 错误:
学号 | 姓名 | 联系方式
001  | 张三 | 电话:123,邮箱:abc@qq.com

✅ 正确:
学号 | 姓名 | 电话 | 邮箱
001  | 张三 | 123  | abc@qq.com
```

### 第二范式(2NF): 消除部分依赖

**要求:** 非主键列完全依赖于主键(不能只依赖主键的一部分)

```
❌ 错误:
学号 | 课程号 | 姓名 | 课程名 | 分数
(学号+课程号是主键,但姓名只依赖学号,课程名只依赖课程号)

✅ 正确:
学生表: 学号 | 姓名
课程表: 课程号 | 课程名
成绩表: 学号 | 课程号 | 分数
```

### 第三范式(3NF): 消除传递依赖

**要求:** 非主键列不依赖于其他非主键列

```
❌ 错误:
学号 | 姓名 | 系号 | 系名
(系名依赖于系号,系号依赖于学号,存在传递依赖)

✅ 正确:
学生表: 学号 | 姓名 | 系号
系表: 系号 | 系名
```

## 四、事务特性(ACID)

### A - Atomicity(原子性)
事务是不可分割的最小单位,要么全部成功,要么全部失败

### C - Consistency(一致性)
事务执行前后,数据保持一致状态

### I - Isolation(隔离性)
多个事务并发执行时,互不干扰

### D - Durability(持久性)
事务一旦提交,对数据的改变是永久的

**记忆口诀:** "原一隔持" 或 "ACID"

## 五、计算机网络

### OSI七层模型(从下到上)

1. **物理层** - 传输比特流(网卡、网线)
2. **数据链路层** - 帧传输(交换机、MAC地址)
3. **网络层** - 路由选择(路由器、IP地址)
4. **传输层** - 端到端通信(TCP、UDP)
5. **会话层** - 建立会话
6. **表示层** - 数据格式转换
7. **应用层** - 用户接口(HTTP、FTP、SMTP)

**记忆口诀:** "物数网传会表应" 或 "Please Do Not Throw Sausage Pizza Away"

### TCP/IP四层模型

1. **网络接口层** = OSI的物理层+数据链路层
2. **网络层** = OSI的网络层
3. **传输层** = OSI的传输层
4. **应用层** = OSI的会话层+表示层+应用层

### 常用协议

| 协议 | 端口 | 作用 |
|------|------|------|
| HTTP | 80 | 网页浏览 |
| HTTPS | 443 | 加密网页浏览 |
| FTP | 21 | 文件传输 |
| SMTP | 25 | 发送邮件 |
| POP3 | 110 | 接收邮件 |
| DNS | 53 | 域名解析 |
| SSH | 22 | 远程登录 |
| Telnet | 23 | 远程登录(不加密) |

### IP地址分类

| 类别 | 范围 | 默认子网掩码 | 用途 |
|------|------|--------------|------|
| A类 | 1.0.0.0 - 126.255.255.255 | 255.0.0.0 | 大型网络 |
| B类 | 128.0.0.0 - 191.255.255.255 | 255.255.0.0 | 中型网络 |
| C类 | 192.0.0.0 - 223.255.255.255 | 255.255.255.0 | 小型网络 |

**私有IP地址:**
- A类: 10.0.0.0 - 10.255.255.255
- B类: 172.16.0.0 - 172.31.255.255
- C类: 192.168.0.0 - 192.168.255.255

## 六、操作系统

### 进程状态

1. **就绪态** - 等待CPU
2. **运行态** - 正在执行
3. **阻塞态** - 等待I/O

### 死锁的四个必要条件

1. **互斥** - 资源不能共享
2. **占有并等待** - 持有资源并等待其他资源
3. **不可剥夺** - 资源不能被强制剥夺
4. **循环等待** - 形成环路

**记忆口诀:** "互占不循"

### 页面置换算法

1. **FIFO** - 先进先出
2. **LRU** - 最近最少使用
3. **OPT** - 最优算法(理论)

## 七、数据结构

### 时间复杂度

| 复杂度 | 名称 | 示例 |
|--------|------|------|
| O(1) | 常数 | 数组访问 |

| O(n) | 线性 | 遍历数组 |
| O(n log n) | 线性对数 | 快速排序 |
| O(n²) | 平方 | 冒泡排序 |

| 算法 | 时间复杂度(平均) | 稳定性 |

| 冒泡排序 | O(n²) | 稳定 |

| 插入排序 | O(n²) | 稳定 |
| 快速排序 | O(n log n) | 不稳定 |
